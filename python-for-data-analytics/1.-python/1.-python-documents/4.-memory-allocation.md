### Memory allocation in Python

In Python, a variable does not store the actual value like a box.

A variable usually stores a **reference** to an object.

That object is stored somewhere in memory.

#### Basic idea

When you assign a value, Python:

1. creates an object
2. stores it in memory
3. connects the variable to that object

```python
a = 10
```

Here, `10` is stored as an object in memory.

`a` refers to that object.

#### How data is stored in a variable

```python
name = "Aman"
age = 21
```

* `"Aman"` is a string object
* `21` is an integer object
* `name` and `age` refer to those objects

So, a variable stores a reference to data.

#### What `id()` does

The `id()` function returns the identity of an object.

It helps us check which object a variable refers to.

```python
x = 50
print(id(x))
```

If two variables refer to the same object, their `id()` can be the same.

```python
x = 50
y = 50
print(id(x))
print(id(y))
```

#### What happens when a value is updated

```python
x = 10
print(id(x))

x = 20
print(id(x))
```

When `x = 20` runs, Python does not change the old object.

It creates or uses another object for `20`.

Now `x` refers to the new object.

So, the `id()` may change.

#### Important point

For many basic data types, changing the value means using a new object.

Examples:

* `int`
* `float`
* `str`

```python
a = "hello"
print(id(a))

a = "hello world"
print(id(a))
```

#### One value and multiple variables

```python
a = 100
b = a

print(id(a))
print(id(b))
```

Here, `a` and `b` can refer to the same object.

#### What is garbage collection

If no variable refers to an object, that object becomes unused.

Python can remove unused objects from memory.

This process is called **garbage collection**.

```python
x = 100
x = 200
```

After this, the old `100` object may become unused.

If no reference is left, Python can clear it from memory.

#### Quick recap

* A variable stores a reference
* Data is stored in memory as objects
* `id()` shows object identity
* Updating a value can point the variable to a new object
* Unused objects are removed by garbage collection

{% hint style="info" %}
Remember this line: In Python, a variable holds a reference to an object.
{% endhint %}
