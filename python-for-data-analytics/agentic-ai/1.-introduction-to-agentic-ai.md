# 1.  Introduction To Agentic AI

Agentic AI is a type of AI that can **think, decide, and act** to complete a task.

It does more than answer one question.

It can break a task into steps, use tools, and move toward a goal.

### Overview

In this lesson, you will learn:

* What Agentic AI means
* How is it different from Generative AI
* How an AI agent works
* where Agentic AI is used
* a simple code example

### Before Agentic AI

To understand Agentic AI, first understand these three terms:

#### Artificial Intelligence

Artificial Intelligence is a system that tries to do tasks that usually need human intelligence.

Examples:

* understanding language
* recognizing images
* making decisions

#### Generative AI

Generative AI creates new content from a prompt.

It can create:

* text
* code
* images

**Example:**

If you ask, _"Write a Python program to add two numbers"_, it can generate the code.

#### Agentic AI

Agentic AI goes one step further.

It not only generates an answer.

It tries to **complete the task**.

It can:

* understand the goal
* plan steps
* use tools
* check results
* try again if needed

{% hint style="info" %}
Generative AI answers. Agentic AI acts.
{% endhint %}

### Generative AI vs Agentic AI

{% tabs %}
{% tab title="Generative AI" %}
* Gives one response at a time
* Focuses on content creation
* Usually waits for the next prompt
* Example: "Write an email"
{% endtab %}

{% tab title="Agentic AI" %}
* Works toward a goal
* Can take multiple steps
* Uses tools and memory
* Example: "Read emails, find meetings, and create a summary"
{% endtab %}
{% endtabs %}

### What is an AI agent

An **AI agent** is a system that can perform tasks on its own to reach a goal.

It usually has:

* a **brain** to reason
* **tools** to do actions
* **memory** to keep context
* a **goal** to finish

### Core parts of Agentic AI

#### 1. Goal

The goal tells the agent what it needs to achieve.

Example:

`Find the top 5 selling products and send a short summary.`

#### 2. Reasoning

The agent thinks about:

* what to do first
* what tool to use
* what result is correct

#### 3. Tools

Tools help the agent interact with the outside world.

Common tools:

* search engine
* calculator
* database
* API
* Python code runner

#### 4. Memory

Memory helps the agent remember previous steps.

This is useful in long tasks.

#### 5. Action

After thinking, the agent performs an action.

Example:

* call an API
* read a file
* send a message

### How Agentic AI works

{% stepper %}
{% step %}
### Understand the task

The agent reads the user request.

It identifies the final goal.
{% endstep %}

{% step %}
### Make a plan

It breaks the work into smaller steps.

This makes complex tasks easier.
{% endstep %}

{% step %}
### Use tools

It selects the right tool for each step.

It may search, calculate, or call an API.
{% endstep %}

{% step %}
### Check the result

It reviews the output.

If the result is weak, it can improve the next step.
{% endstep %}

{% step %}
### Return the final answer

It combines the work into a useful result.
{% endstep %}
{% endstepper %}

### Simple real-life example

Imagine you say:

`Plan a 2-day trip to Jaipur under ₹10,000.`

A normal chatbot may only suggest places.

An agent can do more:

1. search hotel prices
2. check travel options
3. estimate total budget
4. suggest a full plan

That is why Agentic AI feels more like a smart assistant.

### Where Agentic AI is used

Agentic AI is useful in many areas:

* **Customer support** - resolve tickets and draft replies
* **Data analysis** - collect data, clean it, and summarize insights
* **Coding assistants** - write, test, and improve code
* **Research** - search sources and create summaries
* **Business workflows** - automate repeated tasks

### A simple example in code

This example shows the idea of an agent.

It is a basic simulation, not a full production agent.

```python
goal = "Find the sum of 25 and 17"

def think(task):
    return "I should use a calculator"

def use_tool(task):
    return 25 + 17

def respond(result):
    return f"The answer is {result}"

plan = think(goal)
result = use_tool(goal)
final_output = respond(result)

print("Goal:", goal)
print("Plan:", plan)
print("Result:", final_output)
```

#### Output

```python
Goal: Find the sum of 25 and 17
Plan: I should use a calculator
Result: The answer is 42
```

This example shows a simple flow:

* receive a goal
* decide what to do
* use a tool
* return a result

### A more practical agent example

Here is a simple idea of a support agent:

```python
user_question = "What is the price of the Python course?"

def agent(question):
    if "price" in question.lower():
        return "Checking the course details..."
    return "Please ask a course-related question."

print(agent(user_question))
```

In a real system, this agent could:

* check a database
* search documents
* reply with the correct answer

### Why Agentic AI matters

Agentic AI is important because it can handle tasks that need:

* multiple steps
* decision making
* tool usage
* less manual work

It helps save time and improve productivity.

### Benefits of Agentic AI

* Reduces repeated work
* Handles complex workflows
* Works faster on large tasks
* Supports decision making

### Limitations of Agentic AI

Agentic AI is powerful, but it is not perfect.

Some common challenges:

* wrong decisions from unclear prompts
* incorrect tool usage
* incomplete results
* extra cost for many steps

{% hint style="warning" %}
Always review important outputs before using them in real work.
{% endhint %}

### Best practices

Use these rules when building or using agents:

* give a clear goal
* allow only safe tools
* check outputs
* keep humans in the loop for important actions

### Quick recap

Agentic AI is AI that works toward a goal.

It can think in steps, use tools, and take actions.

It is more advanced than a simple prompt-response system.

### Key takeaway

If Generative AI is like a writer, Agentic AI is like an assistant that can **write, search, decide, and act**.
