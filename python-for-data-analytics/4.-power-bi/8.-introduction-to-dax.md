# 8. Introduction to DAX

### What is DAX in Power BI?

**DAX** stands for **Data Analysis Expressions**.

It is a **formula language** used in Power BI to perform calculations and create new information from existing data. It is used to create **calculated columns, measures, and calculated tables** for data analysis.

It works similarly to Excel formulas but is more powerful and is designed for working with large datasets.

<p align="center"><strong>DAX</strong> → Performs calculations after data is loaded.</p>

<p align="center"></p>

### Why is DAX Used?

DAX is used to:

* Calculate **Total Sales**.
* Find **Average Sales**.
* Calculate **Profit** and **Profit Percentage**.
* Compare **Current Year Sales** with **Previous Year Sales**.
* Create **Year-to-Date (YTD)** and **Month-to-Date (MTD)** calculations.
* Create **Rankings** of products or customers.
* Apply Logical conditions such as **IF**, **AND**, and **OR**.
* Create **Calculated Columns**, **Measures**, and **Calculated Tables**.



### How DAX Works



1. **Functions + Operators + Values** — DAX formulas combine built-in functions (like SUM, CALCULATE) with column references, operators, and constants.<br>
2. **Row Context vs Filter Context** — This is DAX's core engine concept:

* _Row context_ = evaluates a formula one row at a time (used in calculated columns).
* _Filter context_ = the set of filters applied by slicers, visuals, or CALCULATE (used in measures).<br>

3. **Evaluation** — When a visual is rendered, Power BI applies the current filter context, then runs the DAX formula against the data model to produce a single value.<br>
4. **Relationships matter** — DAX formulas often pull data across related tables using the model's relationships (e.g., RELATED, RELATEDTABLE).





## Most Commonly Used DAX Functions

### 1. SUM()

Adds all numeric values in a column.

#### Structure of Formula

```
Measure Name = SUM(Table[Column])
```

#### Example

```
Total Sales = SUM(Sales[Sales Amount])
```

#### Formula Breakdown

| Part         | Value        |
| ------------ | ------------ |
| Measure Name | Total Sales  |
| DAX Function | SUM()        |
| Table        | Sales        |
| Column       | Sales Amount |

#### Result

Returns the total sales amount.

#### Use Case

Calculate Total Revenue, Total Profit, Total Quantity.

### 53. COALESCE()

```

```

```

```

#### Use Case

* Prevent blank results in reports and dashboards.
* Improve calculations and visualizations by ensuring a default value is always available.

## Benefits of DAX (Data Analysis Expressions) in Power BI

* **Performs Advanced Calculations** – Easily calculates totals, averages, percentages, profits, and other business metrics.
* **Creates Dynamic Measures** – Automatically updates calculations based on slicers, filters, and user selections.
* **Supports Business Logic** – Applies business rules such as discounts, bonuses, customer categories, and performance ratings.
* **Provides Time Intelligence** – Analyzes data by day, month, quarter, and year using functions like YTD, MTD, and Previous Year.
* **Creates Key Performance Indicators (KPIs)** – Helps monitor business performance with metrics such as Profit Margin, Sales Growth, and Revenue.
* **Improves Data Analysis** – Enables deeper analysis by filtering and calculating data based on different conditions.
* **Works with Multiple Tables** – Retrieves and combines data from related tables using relationships.
* **Creates Calculated Columns** – Generates new columns from existing data without modifying the original source.
* **Creates Powerful Measures** – Performs calculations only when required, making reports more efficient.
* **Provides Dynamic Filtering** – Calculates results according to the filters and slicers selected by users.
* **Reduces Manual Calculations** – Automates repetitive calculations, saving time and reducing errors.
* **Improves Dashboard Performance** – Measures consume less memory than calculated columns, resulting in faster reports.
* **Supports Complex Calculations** – Handles ranking, running totals, percentage calculations, cumulative totals, and conditional logic.
* **Makes Reports Interactive** – Responds instantly to user interactions, making dashboards more engaging.
* **Simplifies Data Modeling** – Works seamlessly with relationships between tables to create meaningful calculations.
* **Enhances Decision Making** – Converts raw data into valuable business insights for informed decisions.
* **Reduces Excel Dependency** – Eliminates the need for complex Excel formulas by performing calculations directly in Power BI.
* **Reusable Calculations** – A single DAX measure can be reused across multiple visuals and report pages.
* **Supports Real-Time Reporting** – Automatically recalculates values whenever the data is refreshed.
* **Improves Report Accuracy** – Uses centralized formulas to ensure consistent calculations throughout the report.
* **Enables Custom Business Metrics** – Allows organizations to create calculations tailored to their specific business requirements.
* **Easy Integration with Visuals** – Measures and calculated columns can be directly used in charts, tables, KPIs, and cards.
* **Handles Large Datasets Efficiently** – Optimized DAX measures help analyze millions of records with good performance.
* **Provides Flexible Filtering** – Functions like `CALCULATE()`, `FILTER()`, and `ALL()` enable advanced filtering scenarios.
* **Essential for Professional Dashboards** – DAX is the foundation for creating advanced, interactive, and professional Power BI dashboards.
