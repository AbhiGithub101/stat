# 16. Joining in Pandas

### Introduction: <a href="#introduction-to-joins-in-pandas" id="introduction-to-joins-in-pandas"></a>

Working on a single table is easy, but what might be challenging is working on multiple tables. You have to think of a lot of criteria, how to join these tables. Pandas offers some joining methods that make this task easy for you. There are four different types of joins in pandas. Let's go through them one by one.

### Types of joins in Pandas : <a href="#types-of-joins-in-pandas" id="types-of-joins-in-pandas"></a>

We’ll take a simple problem from a related marketing brand here. We are given two tables – one that contains data about products and the other that has customer information. We will use these tables to understand how the different types of joins work using Pandas.

### Outer Join : <a href="#outer-join" id="outer-join"></a>

Here’s another interesting task for you. We have to combine both dataframes so that we can find all the products that are not sold and all the customers who didn’t purchase anything from us. We can use a Full Join for this purpose. Full Join, also known as Full Outer Join, returns all those records that have a match in the left or right dataframe. When rows in both the dataframes do not match, the resulting dataframe will have NaN for every column of the dataframe that lacks a matching row. We can perform a full join by just passing the how argument as ‘outer’ to the merge() function:

In \[7]:

```
product.merge(customer,on='Product_ID',how='outer')
```

### Left join <a href="#left-join" id="left-join"></a>

<figure><img src="https://2715416193-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FqrOwR7E0344TGTnSFHoL%2Fuploads%2FbqoOwIdUbJ1XK7Hvnqj5%2Fimage.png?alt=media&#x26;token=2f01aae4-ef2a-4519-8f7e-25f5f046eda2" alt=""><figcaption></figcaption></figure>

Now, let’s say the leadership team wants information about only those customers who bought something from us. You guessed it – we can use the concept of Left Join here.

Left join, also known as Left Outer Join, returns a dataframe containing all the rows of the left dataframe.

All the non-matching rows of the left dataframe contain NaN for the columns in the right dataframe. It is simply an inner join plus all the non-matching rows of the left dataframe filled with NaN for columns of the right dataframe.

Performing a left join is actually quite similar to a full join. Just change the how argument to ‘left’:

In \[8]:

```
product.merge(customer,on='Product_ID',how='left')
```

### Right Join <a href="#right-join" id="right-join"></a>

<figure><img src="https://2715416193-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FqrOwR7E0344TGTnSFHoL%2Fuploads%2FtjBBbHSE7vVlRQdKCmfG%2Fimage.png?alt=media&#x26;token=bfbb3815-d7d8-4113-b9d6-e3e76a1bcd93" alt=""><figcaption></figcaption></figure>

Similarly, if we want to create a table of customers including the information about the products they bought, we can use the right join.

Right join, also known as Right Outer Join, is similar to the Left Outer Join. The only difference is that all the rows of the right dataframe are taken as they are, and only those of the left dataframe that are common to both.

Similar to other joins, we can perform a right join by changing the how argument to ‘right’:

In \[9]:

```
product.merge(customer,on='Product_ID',how='right')
```
