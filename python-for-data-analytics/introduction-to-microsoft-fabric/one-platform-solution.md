# One Platform Solution

Imagine a company called **RetailCorp**.

**Before Fabric**

Different teams use different tools:

| Team            | Tool                              |
| --------------- | --------------------------------- |
| Data Engineers  | Azure data factory/ Azure synapse |
| Data Scientists | Databricks                        |
| Analytics       | Power BI                          |
| Database team   | SQL Warehouse                     |

**Problems:**

* Data duplication
* High cost
* Data silos
* Complex pipelines

**After Fabric**

All teams use **Microsoft Fabric**:

* Data stored in OneLake
* Transformation using Spark / Dataflow
* Warehouse for SQL analytics
* Power BI for reporting

**Before Fabric vs After Fabric**

| Before Fabric       | After Fabric            |
| ------------------- | ----------------------- |
| Multiple tools      | Single unified platform |
| Data duplication    | Single copy in OneLake  |
| Complex integration | Built-in integration    |
| High maintenance    | Simplified management   |
